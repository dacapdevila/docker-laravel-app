Docker Tutorial - Parte 1/2

Primer servicio con estructura monorepo

La idea de esta guia es poder ir paso a paso, avanzando en la creación de la estructura de los microservicios, utilizando Docker.

Pre-requisitos

Github:

  • Crear un repo:

    • Docker Laravel App

  • Subir archivo .gitignore.

  • Crear un nuevo branch, llamado " base ".

  • Agregar la estructura básica:

    mkdir frontend-service cd frontend-service composer create-project laravel/laravel .

  • Subir el contenido al branch " base ".

Creando el Dokerfile del servicio

  • Crear nuevo branch: " base-1-local ".

  • En el caso de que dentro de frontend-service/ no exista un archivo .env, se debe crear:

    cp .env.example .env

  • Editar el archivo .env

    APP_NAME="Frontend Service" APP_ENV=local APP_KEY= APP_DEBUG=true APP_URL=http://localhost:8000 DB_CONNECTION=mysql DB_HOST=db DB_PORT=3306 DB_DATABASE=frontend DB_USERNAME=frontend_user DB_PASSWORD=secret_password # These variables will be used on docker-compose.yml ...

  • Crear Dockerfile:

    touch Dokerfile

  • Copiar el siguiente contenido al Dockerfile:

    # Placed in frontend-service/Dockerfile FROM php:8.2-fpm # Arguments defined in docker-compose.yml ARG user ARG uid # Install system dependecies RUN apt-get update && apt-get install -y \ git \ curl \ libpng-dev \ libonig-dev \ libxml2-dev \ zip \ unzip # Clear cache RUN apt-get clean && rm -rf /var/lib/apt/lists/* # Install PHP extensions RUN docker-php-ext-install pdo_mysql mbstring exif pcntl bcmath gd # Get latest Composer COPY --from=composer:latest /usr/bin/composer /usr/bin/composer # Create system user to run cComposer and Artisan Commands RUN useradd -G www-data,root -u $uid -d /home/$user $user RUN mkdir -p /home/$user/.composer && \ chown -R $user:$user /home/$user # Set working directory WORKDIR /var/www USER $user

Seteando la configuración de Nginx y los database dump files

  • Crear una carpeta en docker-compose/nginx para ubicar toda la configuración de nginx:

    mkdir -p docker-compose/nginx

  • Crear el archivo frontend-service.conf dentro de ese directorio:

    touch docker-compose/nginx/frontend-service.conf

  • Copiar el siguiente contenido en frontend-service.conf:

    server { listen 80; index index.php index.html; # error_log /var/nginx/error.log; # access_log /var/nginx/access.log; root /var/www/public; location / { try_files $uri $uri/ /index.php?$query_string; gzip_static on; } location ~ \.php$ { try_files $uri =404; fastcgi_split_path_info ^(.+\.php)(/.+)$; fastcgi_pass frontend-service:9000; fastcgi_index index.php; include fastcgi_params; fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name; fastcgi_param PATH_INFO $fastcgi_path_info; } location ~ /\.ht { deny all; } }

  • Crear una nueva carpeta dentro de docker-compose/mysql que tendra la inicializacion de acrhcivos de MySQL:

    mkdir docker-compose/mysql

  • Abrir el archivo init_db.sql dentro de la carpeta anterior:

    touch docker-compose/mysql/init_db.sql

  • Copiar el siguiente contenido en init_db.sql:

    DROP TABLE IF EXISTS `places`; CREATE TABLE `places` ( `id` bigint(20) unsigned NOT NULL AUTO_INCREMENT, `name` varchar(255) COLLATE utf8mb4_unicode_ci NOT NULL, `visited` tinyint(1) NOT NULL DEFAULT '0', PRIMARY KEY (`id`) ) ENGINE=InnoDB AUTO_INCREMENT=12 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci; INSERT INTO `places` (name, visited) VALUES ('Berlin', 0),('Budapest', 0),('Cincinnati', 1),('Denver', 0),('Helsinki', 0),('Lisbon', 0),('Moscow', 1),('Nairobi', 0),('Oslo', 1),('Rio', 0),('Tokyo', 0);

Creando el docker-compose.yml

  • Crear el docker-compose.yml en la raíz del proyecto:

    touch docker-compose.yml

  • Copiar el siguiente contenido dentro del docker-compose.yml:

    services: frontend-service: build: args: user: admin uid: 1000 context: ./ dockerfile: Dockerfile image: dockerlaravelapp container_name: docker-laravel-app restart: unless-stopped working_dir: /var/www/ volumes: - ./frontend-service:/var/www networks: - laravel-app db: platform: linux/x86_64 # Mac M1 image: mysql:8.0 container_name: frontend_db restart: unless-stopped environment: MYSQL_DATABASE: ${DB_DATABASE} MYSQL_ROOT_PASSWORD: ${DB_PASSWORD} MYSQL_PASSWORD: ${DB_PASSWORD} MYSQL_USER: ${DB_USERNAME} SERVICE_TAGS: dev SERVICE_NAME: mysql volumes: - ./docker-compose/mysql:/docker-entrypoint-initdb.d ports: - "3307:3306" networks: - laravel-app nginx: image: nginx:alpine container_name: nginx restart: unless-stopped ports: - "8000:80" volumes: - ./frontend-service:/var/www - ./docker-compose/nginx:/etc/nginx/conf.d/ depends_on: - frontend-service networks: - laravel-app networks: laravel-app: driver: bridge

Corriendo la aplicación

  • Para crear la aplicación es necesario correr comandos de Docker:

    docker-compose build frontend-service docker-compose build frontend-service

  • Una vez que termine de crear el servicio, es necesario levantar el servicio con:

    docker-compose up -d

  • Para ver los servicios Docker que esten funcionando:

    docker-compose ps

  • Aún es necesario correr algunos comandos más:

    docker-compose exec frontend-service ls -l

  • Correr compose install:

    docker-compose exec frontend-service rm -rf vendor composer.lock docker-compose exec frontend-service composer install

  • Crear el APP_KEY:

    docker-compose exec frontend-service php artisan key:generate

  • Correr migraciones:

    docker-compose exec frontend-service php artisan migrate

  • Abrir la url:

    http://localhost:8000
