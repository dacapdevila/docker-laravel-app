Docker Tutorial Help

Servidor-2

  • Verificar que esta configurado correctamente (de vuelta en droplet):

    ssh -T git@github.com

  • Clonar el repo:

    git clone -b BRANCH_NAME https://github.com/USERNAME/REPOSITORY.git

  • Levantar los contenedores:

    cd REPOSITORY docker-compose up --build -d docker ps

  • Instalar Certbot:

    sudo apt update sudo apt install certbot python3-certbot-nginx -y

  • Generar los certificados SSL:

    sudo certbot --nginx -d dariocapdevila.com.ar -d www.dariocapdevila.com.ar

Known Issue:

Error response from daemon: driver failed programming external connectivity on endpoint nginx (f4b4570cb5558e6d5a965a5c27b38967080e5bc48a1dcb1b678c627ba8c4d0cf): failed to bind port 0.0.0.0:80/tcp: Error starting userland proxy: listen tcp4 0.0.0.0:80: bind: address already in use

Verificar que esta usando el puerto 80:

sudo lsof -i :80

Detener el proceso que corre en el puerto 80:

sudo systemctl stop nginx
sudo systemctl stop apache2
sudo systemctl stop nginx

Volver a iniciar los contenedores:

docker-compose up --build -d

Otro Known Issue:

docker-compose exec frontend-service rm -rf vendor composer.lock root@docker-app-testing:~/docker-laravel-app# docker-compose exec frontend-service rm -rf vendor composer.lock rm: cannot remove 'composer.lock': Permission denied

Se debe correr el comando como root:

docker-compose exec --user root frontend-service rm -rf vendor composer.lock
